Dynamic Notifications

Dynamic Notifications are useful to alert the user when an element on-screen has been added, deleted, or somehow changed due to a user's action.

Accessible Dynamic Notifications

When sighted users use an application, they can visually obtain information about changes on-screen. Non-sighted users, however, need an aural clue so that they can better understand what is on-screen and navigate the app more easily.

When To Use Dynamic Notifications

There are many scenarios in which Dynamic Notifications are essential to creating an accessible application. One common example is when a user presses a button. If the button changes an element (such as a dynamic image), it may be important to have VoiceOver focus on the element and announce what it changed to. If the button adds or deletes an element (such as the example below), VoiceOver could announce what was added or deleted to the screen.

However, be careful not to add too many notifications! It is important to have a good balance. Make sure that the user is notified and up-to-date, but do not overload the user with notifications. Instead of being informative, these notifications can become disruptive and make the app even more inaccessible. More about this can be seen under "Finding the Balance" in Updating Content.

Implementation of Dynamic Notifications

To add a VoiceOver notification after a function is called in your code, simply use UIAccessibilityPostNotification. There are a few different types of notifications. The most commonly used ones are listed below; however, all of the accessibility notifications can be found here.

UIAccessibilityAnnouncementNotification

UIAccessibilityAnnouncementNotifications are useful if a verbal announcement needs to be made. To add one into a function, simply type UIAccessibilityPostNotification(UIAccessibilityAnnouncementNotification, NSString), where NSString is an NSString of the announcement that should be read by VoiceOver.

UIAccessibilityLayoutChangedNotification

UIAccessibilityLayoutChangedNotifications are useful for changing VoiceOver's focus and for making verbal announcements. The code for this is UIAccessibilityPostNotification(UIAccessibilityLayoutChangedNotification, NSString) or UIAccessibilityPostNotification(UIAccessibilityLayoutChangedNotification, UIView). If the second argument is an NSString or NSMutableString, VoiceOver will announce it. If the second argument is a UIView (such as a UIButton or UILabel), VoiceOver will focus on that element.

UIAccessibilityScreenChangedNotification

UIAccessibilityScreenChangedNotification is similar to UIAccessibilityLayoutNotification in that it will make an announcement or it will focus on an element. To use this, simply type in a function UIAccessibilityPostNotification(UIAccessibilityScreenChangedNotification, NSString) or UIAccessibilityPostNotification(UIAccessibilityScreenChangedNotification, UIView), where UIView could be a UIView, a UIButton, UILabel, etc.
