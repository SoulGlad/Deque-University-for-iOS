Accessibility Traits

Accessibility Traits are yet another useful and important feature in the realm of iOS Accessibility. An accessibility Trait allows you to choose the best description for what an element in your application does.

A trait is like an extra accessibility Label, that lets a user know how your button, link, text field, etc. will react upon interaction. It is important to set up these traits properly so that a user does not get confused when clicking on a "Text Field" opens the web browser.

Implementing Accessibility Traits

Storyboard Editor

Accessibility Traits don't often change while the app is running, so for most people, this method will suffice.

  1. Go to your storyboard editor in XCode
  2. Click on the element you would like to add a Trait for
  3. Open the Identity Inspector in the right bar
  4. Enable Accessibility by checking the box that says "Enabled"
  5. Scroll down to Accessibility and check the box for the trait(s) that you would like to apply

Programmatic Approach

In most apps, Accessibility traits will remain constant at runtime. However, for more advanced applications with a large amount of dynamic content, it may be necessary to change or add Accessibility Traits at runtime. To add or change traits in your code, follow the directions below.

  1. In the Storyboard Editor, turn on accessibility for the item that you want to set a trait for, just like in the first part of this demo
  2. Pull up the header file for your View Controller in the Assistant Editor
  3. Ctrl click and drag from your element to the header file to create an IBOutlet for your element
  4. Now open your implementation file for your ViewController
  5. Use the accessibilityTrait member OR the setAccessibilityTrait function to add the appropriate UIAccessibilityTraits value.

Below are the two ways to set an accessibility Trait.

_DogDisplay.accessibilityLabel = UIAccessibilityTraitButton;
[_DogDisplay setAccessibilityTrait:UIAccessibilityTraitButton];

Discussion

Why you should care about traits

While the impact of traits may seem somewhat subtle and insignificant, it is vastly important in the realm of iOS Accessibility. Traits let the Accessibility API know specific information about your elements that can help VoiceOver and other aspects of the API handle your elements in a clearer and more uniform way.

Tips

  • Make sure that the traits you assign to elements accurately reflect what that element is/does, assigning multiple traits is almost always unnecessary!
  • If the content on the page is dynamic, make sure that your traits change when the function of an element changes
